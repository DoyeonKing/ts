# 剧院智能平台 · 功能使用说明

> 本文档面向演示与日常使用，覆盖四大核心功能模块：**3D 选座**、**知识图谱**、**AI 推荐**、**AI 文创生成**。

---

## 一、3D 选座

### 功能概述

在选座时提供剧场三维透视预览，帮助观众在购票前直观感受每个座位的实际视角与遮挡情况，避免因视野不佳导致的观演体验落差。

### 使用步骤

**第一步：进入选座页面**

在演出详情页选择场次后，点击"选座"即可进入 3D 选座页面。页面顶部显示演出名称、场次时间、场馆信息，以及当前已加入参考的座位数量。

**第二步：浏览全局视角**

初始进入「全局俯瞰」模式：

- **单指拖动**：旋转视角，360° 环视整个剧场
- **双指捏合 / 张开**：缩放远近
- 舞台区域标注有 `PROSCENIUM STAGE` 标识
- 座位按状态用颜色区分：**红色** = 可选，**金色** = 已选，**灰色** = 已售

**第三步：点击座位，切换第一人称视角**

点击任意可售座位后，镜头将平滑过渡到该座位的真实视线高度，模拟坐在该位置时的观看角度。信息面板随即弹出，显示：

| 信息项 | 说明 |
|---|---|
| 座位编号 | 排号 + 列号 |
| 视野评语 | 如"舞台正中心，视野极佳" |
| 推荐指数 | 0–100 分，综合距离与角度计算 |

**第四步：切换遮挡模拟模式**

在控制面板中选择前排观众的身高模式，实时观察遮挡效果：

| 模式 | 说明 |
|---|---|
| 无遮挡 | 清空前排，评估纯粹视线角度 |
| 正常就座 | 模拟普通身高观众入座后的遮挡 |
| 高个就座 | 前排加高比例模型，模拟极端遮挡场景 |

在第一人称视角下可**单指拖动**，自由环顾左右，感受视线范围。

**第五步：加入参考 & 完成选座**

- 点击「加入参考座位」：将当前座位记录至参考列表
- 点击「返回全局」：回到俯瞰视角继续浏览其他座位
- 底部栏实时展示已选座位，点击「完成」进入支付流程

> **提示**：页面右上角标注"仅供参考"，3D 视角为比例模拟，实际灯光和舞台布景以现场为准。若设备不支持 WebGL，页面将自动退回 2D 座位图，功能不受影响。

## 二、知识图谱

### 功能概述

以三维浮动球体图的形式，呈现剧目、演员、场馆、剧场术语之间的关联关系，帮助用户探索剧场知识体系，发现感兴趣的内容脉络。

### 使用步骤

**第一步：进入知识图谱页面**

在首页或发现页点击"知识图谱"入口。进入后，图谱将从服务端拉取节点与边数据并自动布局，期间显示加载动画。

**第二步：浏览与交互**

图谱加载完成后进入自动旋转状态：

- **拖动画布**：旋转整个图谱，任意角度查看
- **节点大小**：节点越大，与其他节点的关联数越多（重要度越高）
- **颜色图例**（页面底部）：

| 颜色 | 节点类型 |
|---|---|
| 蓝色 | 剧目 |
| 橙色 | 演员 |
| 绿色 | 术语 |
| 紫色 | 场馆 / 其他 |

**第三步：搜索定位**

在顶部搜索栏输入关键词（如演员名、剧目名、术语），图谱将高亮匹配节点并将镜头聚焦至该节点，其关联边同步加亮显示。

**第四步：按类型筛选**

点击顶部筛选标签（剧目 / 演员 / 术语 / 全部），可以隐藏不感兴趣的节点类型，让图谱聚焦于特定维度的关系网络。

**第五步：点击节点查看详情**

点击任意节点后：

- 图谱停止旋转
- 该节点的所有关联边高亮
- 底部弹出**详情卡片**，显示：
  - 节点名称与类型
  - 简介描述
  - 关联节点数量

卡片内提供两个操作：

| 按钮 | 功能 |
|---|---|
| 进入详情 | 跳转至该实体的独立详情页 |
| 以此为中心 | 以当前节点为原点，重新构建局部关系图谱 |

## 三、AI 推荐

### 功能概述

基于 RAG（检索增强生成）架构，将用户的自然语言需求解析为结构化约束，结合真实在售票务数据与大模型分析，生成个性化的演出推荐方案，并对推荐逻辑全程透明展示。

### 使用步骤

**第一步：描述观演需求**

在输入框中用自然语言描述需求，可以包含任意组合的条件，例如：

> "下周末在北京，两个人看，预算不超过 600，最好是话剧或音乐剧"

页面顶部也提供**快捷示例标签**（点击即可填入），适合首次体验：

- 北京周末场，300 以内
- 上海亲子剧，适合带小孩
- 经典话剧，演员阵容强

**第二步：启动推荐**

点击「开始推荐」后，系统将依次执行以下步骤（界面同步展示进度）：

```
01 意图解析    →  提取城市、预算、剧种、档期等约束
02 特征生成    →  生成"台词张力强""群像戏"等软标签
03 候选召回    →  从剧目库检索匹配候选
04 事实过滤    →  结合真实票价与在售状态精确筛选
05 方案输出    →  综合排序，生成推荐理由
```

**第三步：查看推荐结果**

结果分为三个区块：

**① 意图分析卡**

展示系统对需求的理解，包括：解析出的城市、预算上限、时间偏好，以及意图置信度百分比。

**② 算法追踪**

以数字面板形式显示：

- 召回命中数：初始候选池大小
- 过滤淘汰数：因票价/场次不符被剔除的数量
- 事实约束强度：真实票务数据对结果的影响权重

> 此区块帮助用户理解"为什么没有推荐某部戏"。

**③ 推荐方案列表**

每条推荐卡片包含：

| 信息 | 说明 |
|---|---|
| 匹配评分 | 综合匹配度（百分制） |
| 演出信息 | 剧目名称、场馆、价格区间 |
| 推荐理由 | 一句话概括匹配逻辑 |
| 分析依据 | 展开后显示软标签、演员特质、专业评语等细节 |

**第四步：调整需求**

若结果为空，系统将提示可能的原因（如预算过低或档期范围太窄）并给出调整建议，修改输入后可再次发起推荐。

## 四、AI 文创生成

### 功能概述

以演出为主题，通过大模型（通义千问）生成文创周边的设计方案与宣传文案，再调用图像生成模型（DashScope）将方案可视化，支持周边商品与海报两种输出形式。

### 使用步骤

**第一步：填写创意主题**

在输入框中描述想要的文创风格与方向，例如：

> "《茶馆》主题，新中式风格，适合做帆布袋和马克杯"

> "《哈姆雷特》，极简黑白，高级感海报"

**第二步：选择输出类型**

通过顶部切换标签选择生成目标：

| 类型 | 适用场景 |
|---|---|
| 周边商品 | T恤、帆布袋、马克杯、徽章等实体文创 |
| 宣传海报 | 演出官方海报、活动宣传图 |

**第三步：生成创意方案**

点击「生成方案」，系统进行第一阶段生成，输出包含：

- **需求分析**：对主题风格的理解与设计定位
- **平面设计提示词**：指导排版、色彩、构图的专业描述
- **成品展示提示词**：用于图像生成的完整英文 Prompt
- **周边建议清单**：每件单品附带设计亮点与推荐文案语

**第四步：生成图像**

方案生成后，点击「生成图像」，系统将方案中的统一 Prompt 发送至图像生成 API，约数秒后在页面展示效果图。

**第五步：查看与使用**

- 长按图片可保存至相册
- 周边清单中的文案（如 slogan）可直接复制用于实际设计沟通
- 若对效果不满意，可修改输入主题后重新生成

> **提示**：若图像生成服务未配置或调用异常，页面将显示"本地降级"标识，此时仍可正常查看文案方案；图像功能待 API Key 配置生效后自动恢复。
