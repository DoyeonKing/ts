# 解决 @oasis-engine 报错（ENOENT）

报错原因：当前打开的微信小程序构建产物是**旧版本**，里面的 JS 仍在 `require('@oasis-engine/...')`，而项目已改为 Three.js，不再安装 Oasis，所以会找不到文件。

**必须用当前源码重新构建一次，并只构建 threejs-miniprogram。**

---

## 步骤（请严格按顺序）

### 1. 关闭微信开发者工具  
完全退出，不要只关项目。

### 2. 删除小程序输出目录  
在 **PowerShell** 或 **命令提示符** 里执行（把路径改成你的项目实际路径）：

```powershell
cd D:\Desktop\thearter\version_01\uni_app
Remove-Item -Recurse -Force .\unpackage\dist\dev\mp-weixin -ErrorAction SilentlyContinue
```

若提示“正在使用”或“拒绝访问”，说明还有程序占用（微信开发者工具、HBuilderX 预览等），请关掉后再删一次。

### 3. 重新编译小程序  
任选一种方式：

- **方式 A**：在 HBuilderX 中 **运行 → 运行到小程序模拟器 → 微信开发者工具**  
- **方式 B**：在项目根目录执行  
  ```powershell
  npm run build:mp-weixin
  ```

这样会重新生成 `unpackage/dist/dev/mp-weixin`，且**只包含当前源码**（已无 Oasis，只有 threejs-miniprogram）。

### 4. 准备 npm（小程序目录里的 package.json）  
在项目根目录执行：

```powershell
npm run prepare-mp-npm
```

### 5. 在微信开发者工具里构建 npm  
1. 用微信开发者工具打开目录：`D:\Desktop\thearter\version_01\uni_app\unpackage\dist\dev\mp-weixin`  
2. 菜单 **工具 → 构建 npm**  
3. 构建完成后可 **项目 → 重新打开此项目** 再预览。

此时只会安装 **threejs-miniprogram**，不会再出现 @oasis-engine 相关报错。

---

## 若仍报 @oasis-engine

1. 确认已**完全关闭**微信开发者工具后，再执行步骤 2 删除 `mp-weixin`。  
2. 确认步骤 3 是在**当前项目**下执行的（没有改错目录）。  
3. 确认步骤 4 执行成功（`unpackage/dist/dev/mp-weixin` 下能看到 `package.json` 和 `node_modules`，且 `package.json` 里只有 `threejs-miniprogram`，没有 `oasis-engine`）。
