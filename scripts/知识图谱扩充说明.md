# 知识图谱扩充说明

这次我已经额外补了一套可以直接执行的扩展素材，目标是解决“知识图谱内容太少”的问题。

## 新增文件

- `scripts/knowledge-graph-seed-expanded.sql`
  - 直接给 MySQL 用
  - 已包含清空旧数据、重置自增、插入节点、插入关系
- `scripts/knowledge-graph-neo4j-expanded.cypher`
  - 直接给 Neo4j Browser 用
  - 已包含清空图、创建节点、创建关系

## 扩充后的内容规模

当前这套扩展版大致包括：

- 剧目：13 个
- 演员：6 个
- 标签：11 个
- 术语：15 个
- 场馆：5 个
- 关系：演员参演、剧目标记、剧目术语、剧目场馆、演员场馆、相似剧目、术语关联 等多种类型

比原始种子数据明显更丰富，更适合前端知识图谱展示、搜索和答辩演示。

## MySQL 使用方式

### 1）如果表已经存在
直接执行：

```bash
mysql -u root -p theater_db < scripts/knowledge-graph-seed-expanded.sql
```

### 2）如果表还不存在
先执行：

```bash
mysql -u root -p < scripts/knowledge-graph-schema.sql
```

再执行：

```bash
mysql -u root -p theater_db < scripts/knowledge-graph-seed-expanded.sql
```

### 3）Navicat / DBeaver
直接打开并运行：

- `scripts/knowledge-graph-schema.sql`
- `scripts/knowledge-graph-seed-expanded.sql`

## Neo4j 使用方式

打开 Neo4j Browser，把 `scripts/knowledge-graph-neo4j-expanded.cypher` 全部复制进去运行即可。

脚本第一句是：

```cypher
MATCH (n) DETACH DELETE n;
```

意思是会先清空旧图数据；如果你不想清空，把这一句删掉再执行。

## 说明

### 当前后端兼容性
你现在 SpringBoot 后端仍然是基于 MySQL 的 `knowledge_node` / `knowledge_edge` 两张表在跑，所以：

- 如果你项目当前要“马上可用”，优先用 MySQL 的扩展 SQL。
- 如果你答辩或演示时想展示“真正的图数据库方案”，就再用 Neo4j 的 `.cypher` 文件。
