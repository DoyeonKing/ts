# 知识图谱数据说明

## 你是不是已经做好数据库了？

**是的，知识图谱相关的数据库已经设计好了。**

- **表结构**：`knowledge_node`、`knowledge_edge` 由 SpringBoot JPA 实体定义
- **建表方式**：`application.yml` 中 `ddl-auto: update`，启动 SpringBoot 后 Hibernate 会自动创建/更新表
- **数据初始化**：`KnowledgeGraphService.initSeedData()` 已在代码中写好，调用 `POST /api/knowledge-graph/init` 即可

---

## 两种使用方式

### 方式一：用 SpringBoot 接口（推荐）

1. 创建空库：`CREATE DATABASE theater_db CHARACTER SET utf8mb4;`
2. 修改 `application.yml` 中的 MySQL 用户名、密码
3. 启动 SpringBoot：`mvn spring-boot:run`
4. 调用初始化：`POST http://localhost:8080/api/knowledge-graph/init`

数据会由 Java 代码写入，无需手动执行 SQL。

---

### 方式二：别人建好库后，直接用 SQL 导入

**步骤 1**：若表还不存在，先执行建表脚本：

```bash
mysql -u root -p < scripts/knowledge-graph-schema.sql
```

**步骤 2**：导入种子数据：

```bash
mysql -u root -p theater_db < scripts/knowledge-graph-seed.sql
```

或在 Navicat、DBeaver 等工具中依次执行 `knowledge-graph-schema.sql`、`knowledge-graph-seed.sql`。

---

## 脚本内容

`knowledge-graph-seed.sql` 包含：

- 28 个节点：剧目 6、演员 3、标签 7、术语 10、场馆 2
- 若干关系边：演员-剧目、剧目-标签、剧目-术语、剧目-场馆、剧目-剧目相似、演员-场馆、术语-术语相关

与 `KnowledgeGraphService.initSeedData()` 中的逻辑一致。
