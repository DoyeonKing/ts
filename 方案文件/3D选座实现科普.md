# 3D 选座界面实现科普（零基础版）

本文档专门解释：`uni_app/pages/seat-selection-3d/seat-selection-3d.vue` 是怎么做出来的，每段代码在干什么。

---

## 1. 先理解这个页面在做什么

这个页面本质是一个“剧场空间模拟器”，不是支付系统。

它做了三件事：

1. **构建一个3D剧场空间**（舞台、灯光、座椅）
2. **让用户在空间里选座位并切换视角**
3. **根据座位位置给一个观演参考指数**（便于比较不同座位）

所以你答辩可以说：
> 页面采用“3D优先 + 2D保底”策略，保证演示稳定性。

## 3. 座位数据是怎么来的

不是从后端逐座位拉的，而是前端按规则生成一个标准化剧场座位网格。

```145:173:uni_app/pages/seat-selection-3d/seat-selection-3d.vue
function buildSeatList() {
  const list = []
  const soldSet = new Set(SOLD_SEATS)
  for (let r = 0; r < ROWS.length; r++) {
    for (let c = 0; c < COLS; c++) {
      const offset = c - (COLS - 1) / 2
      const curve = Math.abs(offset) * 0.16 * (1 + r * 0.03)
      const seatId = ROWS[r] + (c + 1)
      const rakeHeight = r * 0.32
      list.push({
        row: ROWS[r],
        rowIndex: r,
        col: c + 1,
        seatId,
        status: soldSet.has(seatId) ? 1 : 0,
        x: offset * 1.28,
        y: 0.18 + rakeHeight,
        z: -9.6 + r * 1.52 + curve
      })
    }
  }
  return list
}
```

重点解释：
- `x/y/z`：每个座位在三维空间中的坐标
- `rakeHeight`：越往后排，地面越高（模拟真实剧场坡度）
- `curve`：边缘座位会有一点弧度（看起来更像扇形观众席）
- `status`：是否是不可选座位（卖出/禁用）

你可以把 Three.js 简化理解为：
> Scene（舞台） + Camera（观众眼睛） + Renderer（投影仪）

## 8. 第一视角怎么计算

```584:597:uni_app/pages/seat-selection-3d/seat-selection-3d.vue
const eyeY = seat.y + 1.24 + VIEW_MODES[this.viewMode].height
const eyeZ = seat.z + 0.24
this.camera.position.set(seat.x, eyeY, eyeZ)
this.camera.lookAt(seat.x, eyeY + VIEW_MODES[this.viewMode].pitch * 5.2, STAGE_CENTER.z)
```

这段意思：
- 把相机放在“观众坐下后头部大概高度”
- 看向舞台中心
- 不同模式调整 `height/pitch`

所以用户会感觉“我真的坐在这个座位看舞台”。

## 9. 推荐指数是怎么来的

```222:228:uni_app/pages/seat-selection-3d/seat-selection-3d.vue
const rowScore = 100 - this.currentSeat.rowIndex * 5
const sidePenalty = Math.abs(this.currentSeat.col - (COLS + 1) / 2) * 3
return clamp(Math.round(rowScore - sidePenalty + VIEW_MODES[this.viewMode].delta), 52, 98)
```

这不是机器学习模型，而是可解释规则：
- 越靠后，分越低
- 离中轴越远，分越低
- 遮挡模式会扣分（`delta`）

优点：
- 简单
- 可解释
- 答辩时好讲

## 10. 交互与稳定性设计

### 10.1 手势交互
- 单指拖动：环绕或第一视角环顾
- 双指缩放：调整俯瞰距离

### 10.2 失败降级
- Three.js 初始化失败 -> 自动显示 fallback 2D 座位图

### 10.3 资源释放
页面卸载时释放 renderer 和动画，避免内存泄漏。

## 12. 重点术语速记

- `Scene`：三维场景容器
- `Camera`：观察视角
- `Renderer`：渲染器
- `Mesh`：3D实体（几何体+材质）
- `Raycaster`：点击检测（射线拾取）
- `fallback`：降级兜底方案
- `viewMode`：无遮挡/坐姿/高个三种观察模式
